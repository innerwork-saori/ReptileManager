# ReptileManager 需求規格書初稿

## 專案概述

ReptileManager 是一個以 PWA 形式提供的爬蟲專用日常管理工具，主要供單一使用者自用，支援多隻爬蟲管理、離線使用、快速記錄與本機資料保存。[cite:11][cite:18]

本 App 的核心情境是：使用者可以管理多隻爬蟲的基本資料、健康資料、餵食紀錄、投藥紀錄、環境監測與今日待辦，並可透過每隻寵物專屬的 QR Code 快速開啟該爬蟲的詳細頁面進行記錄。[cite:28]

## 部署可行性

本系統可以部署在 Netlify，因為 Netlify 適合靜態網站與 SPA/PWA 類型專案，並支援以 `_redirects` 或 `netlify.toml` 設定前端路由重寫規則。[cite:70][cite:73]

若前端採用 SPA 路由，需設定類似 `/* /index.html 200` 的 rewrite 規則，否則使用者直接開啟深層網址，例如爬蟲詳細頁 deep link，可能會遇到 404 問題。[cite:70][cite:61]

Netlify 也可透過 `_headers` 或 `netlify.toml` 設定快取標頭，這對 PWA 的 HTML、靜態資源與 service worker 更新策略很重要。[cite:68][cite:71]

## 產品定位

- App Name：ReptileManager。
- 平台形式：PWA。[cite:11][cite:18]
- 使用對象：單一使用者，自用為主。
- 使用模式：一份本機資料可管理多隻爬蟲。
- 資料儲存：client-side 為主，支援匯出 JSON 備份。[cite:11][cite:18][cite:55]
- 提醒方式：第一版以開啟 App 時在首頁顯示提醒為主，不承諾 App 完全關閉後仍準時跳出背景通知。[cite:22][cite:25][cite:17]

## 產品目標

### 主要目標

1. 建立每隻爬蟲的基本資料與品種管理。
2. 記錄餵食、投藥、脫皮、環境監測與待辦完成狀態。
3. 依不同爬蟲種類顯示對應欄位與預設待辦。
4. 讓使用者透過 QR Code 快速進入指定爬蟲頁面。
5. 讓資料可於本機離線保存，並可匯出為 JSON 備份。[cite:11][cite:18][cite:55]

### 非目標

以下內容不納入第一版 MVP：

- 多使用者協作。
- 雲端同步。
- 需 server 才能保證的背景推播提醒。
- App 內建相機掃碼功能。
- 複雜醫療欄位，例如醫院、醫師、費用、附件照片。
- 哺乳類相關流程，例如散步、排泄、美容。

## 使用情境

### 核心情境

- 使用者在首頁查看所有爬蟲今日狀態與待辦。
- 使用者點入某一隻爬蟲詳細頁，查看餵食、投藥、健康、環境與待辦資料。
- 使用者掃描爬蟲專屬 QR Code，直接進入該爬蟲詳細頁，再選擇要新增餵食、投藥、脫皮或環境紀錄。[cite:28]
- 使用者建立療程與待辦規則，系統在 App 開啟時自動計算目前待處理事項。[cite:22][cite:25]
- 使用者定期將所有資料匯出為 JSON 檔案保存。[cite:55][cite:57]

## 功能需求

### 1. 爬蟲基本資料

系統需支援多爬蟲管理，每隻爬蟲至少包含以下基礎欄位：

- reptileId（系統唯一識別碼）
- 名稱
- 物種
- 品種
- 性別
- 出生日期或年齡
- 照片
- 備註
- 飼養箱 / 位置名稱（選填）
- QR Code 對應網址

系統需支援兩層分類結構：物種 > 品種。

### 2. 健康資訊

系統需支援以下健康資訊：

- 體重歷史曲線：每次量測需保留日期、數值與備註。
- 過敏資訊：基礎文字欄位。
- 慢性病資訊：基礎文字欄位。
- 看診紀錄：先提供基礎欄位，例如日期、項目、備註。
- 脫皮紀錄：日期、狀態、備註。

第一版不要求進階醫療附件管理。

### 3. 餵食紀錄

系統需支援手動新增餵食紀錄，單筆紀錄至少包含：

- 爬蟲
- 餵食時間
- 食物類型
- 食量
- 備註

首頁需能顯示每隻爬蟲距離上一次餵食多久，作為快速判讀資訊。

### 4. 投藥管理

系統需同時支援「事後投藥紀錄」與「療程設定」。

#### 4.1 投藥紀錄

單筆投藥紀錄至少包含：

- 爬蟲
- 投藥時間
- 藥品名稱
- 劑量
- 備註

#### 4.2 療程設定

療程需至少支援以下週期規則：

- 每天
- 每 X 小時
- 每週幾次

系統需能根據療程規則，在 App 開啟時判斷是否有待執行投藥提醒。[cite:22][cite:25]

### 5. 環境管理

系統需支援爬蟲飼養常見環境管理項目：

- 溫度紀錄
- 濕度紀錄
- UVB / 燈照管理
- 更換墊材紀錄

#### 5.1 溫濕度紀錄

單筆紀錄至少需包含：

- 爬蟲
- 紀錄時間
- 溫度
- 濕度
- 備註

#### 5.2 UVB / 燈照管理

需可記錄燈具名稱、開始日期、預計更換日與備註。

#### 5.3 墊材更換紀錄

需可記錄更換日期、類型與備註。

### 6. 今日待辦

系統需支援由規則自動產生待辦事項，使用者可自行設定規則。

#### 6.1 待辦規則類型

- 固定排程，例如每天 8:00 餵食、每週日清潔、更換水盆。
- 條件觸發，例如餵食後 2 小時投藥、脫皮後觀察狀態。

#### 6.2 待辦實例

系統需在首頁顯示今日待辦，並可標示完成狀態。

單筆待辦至少需包含：

- 爬蟲
- 待辦類型
- 預計時間
- 狀態（待完成 / 已完成 / 已略過）
- 備註

### 7. 爬蟲專用預設管理項目

系統需內建適合爬蟲管理的預設項目與欄位，至少包含：

- 餵食
- 投藥
- 體重
- 脫皮
- 溫度
- 濕度
- UVB / 燈照
- 更換墊材
- 清潔 / 換水

### 8. QR Code 快速進入

每隻爬蟲需對應一個專屬 QR Code，且一隻爬蟲僅需一個 QR Code。

QR Code 掃描後應導向該爬蟲詳細資料頁，而非直接進入新增某一種紀錄頁，讓使用者進頁後再自行選擇下一步操作。[cite:28]

第一版採外部相機或外部掃碼工具掃描 QR Code 後開啟頁面，不納入 App 內建掃碼器。[cite:31][cite:35]

### 9. 資料匯出與備份

系統需支援將本機資料匯出為 JSON 檔案，供使用者備份保存。[cite:55][cite:57]

若瀏覽器支援，可考慮透過 Web Share API 呼叫系統分享面板；若不支援，則至少需提供檔案下載功能。[cite:46][cite:55]

## 非功能需求

### 1. 儲存方式

本系統以 client-side 儲存為主。

由於本專案資料包含多隻爬蟲、歷史紀錄、規則型待辦與療程設定，主資料儲存建議採用 IndexedDB，而非 localStorage；localStorage 僅建議用於少量 UI 狀態資料。[cite:11][cite:18]

### 2. 離線能力

系統應支援離線開啟已安裝的 PWA，並可讀取本機已存在資料。[cite:11][cite:18]

### 3. 部署方式

系統應可作為靜態前端專案部署於 Netlify，並配置 PWA 必要資源、路由重寫與快取策略。[cite:70][cite:68]

### 4. 更新策略

因 PWA 與 service worker 可能受瀏覽器快取影響，需設計適當快取策略：

- HTML 入口頁應設定需重新驗證。
- 帶 hash 的靜態資源可長時間快取。
- 發現新版時應提示使用者重新整理。[cite:71][cite:68]

## 使用流程

### 流程一：新增爬蟲

1. 使用者建立爬蟲基本資料。
2. 系統產生 reptileId。
3. 系統建立對應 deep link。
4. 系統產生爬蟲專屬 QR Code。
5. 使用者可於詳細頁查看並存檔 QR Code。

### 流程二：掃碼快速記錄

1. 使用者用外部相機掃描爬蟲 QR Code。
2. 系統開啟對應 deep link。
3. App 導向該爬蟲詳細頁。
4. 使用者選擇要新增餵食、投藥、脫皮或環境紀錄。[cite:28]

### 流程三：開啟 App 查看今日待辦

1. 使用者開啟 App。
2. 系統讀取本機資料與規則。
3. 系統計算今日待辦、待投藥項目與提醒。
4. 首頁顯示所有爬蟲總覽與待處理事項。[cite:22][cite:25]

## MVP 範圍

### 必做

- 多爬蟲管理。
- 爬蟲基本資料。
- 體重歷史曲線。
- 餵食紀錄。
- 投藥紀錄。
- 療程規則（每天、每 X 小時、每週幾次）。
- 今日待辦規則（固定排程、條件觸發）。
- 脫皮紀錄。
- 溫濕度紀錄。
- UVB / 燈照管理。
- 更換墊材紀錄。
- 所有爬蟲總覽首頁。
- 每隻爬蟲 1 個 QR Code。
- JSON 匯出備份。
- IndexedDB 本機儲存。
- Netlify 部署。

### 可延後

- JSON 匯入還原。
- 雲端同步。
- App 內建掃碼。
- 背景推播提醒。
- 列印版 QR Code 樣板。
- 更完整看診欄位。
- 圖表式健康與環境分析。
- 多飼養空間管理。

## 風險與限制

- 若僅採 client-side 儲存，使用者清除瀏覽器資料後，本機資料可能遺失，因此 JSON 匯出備份屬重要功能。[cite:18][cite:24][cite:55]
- PWA 提醒在不同平台支援度不一，第一版以「開 App 時首頁提醒」較穩定。[cite:22][cite:25][cite:17]
- 若使用 deep link 跳入 SPA 詳細頁，部署時必須正確配置 Netlify rewrite 規則。[cite:70][cite:61]
- 若未來導入 service worker 快取，需處理版本更新與舊快取失效問題。[cite:71][cite:68]

## 驗收重點

1. 使用者可建立多隻爬蟲，並保存基礎資料。
2. 首頁可查看所有爬蟲總覽。
3. 使用者可為每隻爬蟲新增餵食與投藥紀錄。
4. 首頁可顯示距離上次餵食多久。
5. 使用者可建立療程與待辦規則。
6. 系統可記錄脫皮、溫濕度、UVB 與墊材更換資料。
7. 系統可在開啟 App 時自動生成今日待辦與提醒。
8. 每隻爬蟲皆可產生 QR Code，掃描後可進入該爬蟲詳細頁。[cite:28]
9. 使用者可匯出 JSON 檔案備份資料。[cite:55][cite:57]
10. 系統可成功部署於 Netlify，且 deep link 可正常運作。[cite:70][cite:73]
